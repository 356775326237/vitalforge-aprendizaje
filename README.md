Landing Page de VitalForge

Descripción General del Proyecto

Este proyecto consiste en una landing page responsiva para una marca ficticia de suplementos deportivos llamada VitalForge. El objetivo no era únicamente crear un sitio web visualmente atractivo, sino también demostrar habilidades de resolución de problemas, investigación, uso de herramientas modernas de desarrollo y capacidad de aprendizaje rápido.

Demo en Vivo: https://vitalforge-aprendizaje.vercel.app/

⸻

1. ¿Cómo desglosé el problema antes de empezar?

Antes de escribir cualquier línea de código, dividí el proyecto en partes más pequeñas y manejables.

Primero identifiqué los requisitos principales de una landing page para una marca de suplementos:

* Barra de navegación
* Sección principal (Hero)
* Muestra de productos
* Testimonios de clientes
* Formulario de suscripción
* Sección de analítica
* Diseño responsivo

Después de definir estas secciones, organicé la estructura del proyecto utilizando componentes reutilizables de React. Esto me permitió trabajar en cada sección de manera independiente.

También planifiqué la experiencia del usuario desde el inicio hasta el final de la página para asegurar que los visitantes pudieran comprender fácilmente los productos, conocer sus beneficios y suscribirse.

⸻

2. ¿Qué herramientas elegí y por qué? ¿Qué alternativas consideré?

Herramientas Seleccionadas

Next.js

Elegí Next.js porque proporciona un framework moderno basado en React con excelente rendimiento, sistema de rutas y facilidad de despliegue.

React

React me permitió crear componentes reutilizables y organizar la interfaz de manera eficiente.

TypeScript

TypeScript mejora la confiabilidad del código y ayuda a prevenir errores comunes durante el desarrollo.

Tailwind CSS

Tailwind permitió desarrollar la interfaz de manera rápida y mantener un diseño consistente en todo el proyecto.

Git y GitHub

Utilizados para el control de versiones, seguimiento de cambios y administración del código fuente.

Vercel

Elegido porque se integra perfectamente con Next.js y ofrece un proceso de despliegue simple y eficiente.

⸻

Alternativas Consideradas

* HTML, CSS y JavaScript.
* Bootstrap.
* Firebase Hosting.
* React tradicional sin Next.js.

Finalmente elegí Next.js y Tailwind porque ofrecían el mejor equilibrio entre experiencia de desarrollo y facilidad de despliegue.

⸻

3. ¿Qué desafíos tuve y cómo los resolví?

Diseño Responsivo

Uno de los principales desafíos fue asegurar que la página se visualizara correctamente en dispositivos móviles.

Inicialmente algunos textos eran demasiado grandes y ciertas secciones no se adaptaban correctamente a pantallas pequeñas.

Para solucionarlo:

* Ajusté los tamaños de texto de forma responsiva.
* Probé la página en diferentes tamaños de pantalla.
* Utilicé las utilidades responsivas de Tailwind CSS.
* Refiné los espacios y la distribución de los elementos.

Proceso de Despliegue

Otro desafío fue comprender el funcionamiento de Git, GitHub y Vercel.

Para resolverlo:

* Creé un repositorio en GitHub.
* Aprendí el flujo de trabajo de Git (add, commit y push).
* Conecté GitHub con Vercel.
* Verifiqué el despliegue automático después de cada actualización.

Integración de Analítica

La integración de analítica requirió comprender cómo funciona Vercel Analytics y cómo conectarlo correctamente a la aplicación.

Después de investigar la documentación oficial y realizar pruebas, logré integrar el seguimiento de visitantes correctamente.

4. ¿Cómo utilicé herramientas de IA? Dame ejemplos concretos.

Las herramientas de inteligencia artificial fueron utilizadas como asistentes de aprendizaje, investigación y productividad durante el desarrollo del proyecto. Mi enfoque fue utilizarlas para acelerar el proceso de aprendizaje y resolver problemas específicos, manteniendo siempre el control sobre las decisiones de diseño y desarrollo.

ChatGPT

Utilicé ChatGPT principalmente como apoyo técnico y educativo para:

* Comprender conceptos de React, Next.js y TypeScript.
* Aprender el flujo de trabajo con Git y GitHub.
* Resolver errores durante el desarrollo.
* Mejorar la estructura y organización de los componentes.
* Implementar diseño responsivo para dispositivos móviles.
* Comprender el proceso de despliegue en Vercel.
* Integrar Vercel Analytics.
* Revisar y mejorar la documentación del proyecto.

Ejemplo

Durante el desarrollo encontré dificultades al conectar componentes, manejar imágenes y adaptar la interfaz a dispositivos móviles. Utilicé ChatGPT para entender las causas de los problemas y aplicar soluciones de manera progresiva mientras aprendía los conceptos detrás de cada cambio.

Ejemplo

Durante la configuración de GitHub y Vercel utilicé ChatGPT para comprender los comandos de Git, el proceso de despliegue y las mejores prácticas para publicar una aplicación web en producción.

Gemini

Utilicé Gemini para la generación de imágenes de los productos mostrados en la landing page.

Ejemplo

Generé imágenes conceptuales de los suplementos VitalForge utilizando prompts detallados enfocados en:

* Apariencia premium.
* Consistencia visual entre productos.
* Diferenciación mediante colores.
* Estética moderna relacionada con nutrición deportiva.

Posteriormente seleccioné, ajusté y optimicé las imágenes para integrarlas dentro de la interfaz del proyecto.
